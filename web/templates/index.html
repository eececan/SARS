About this Project

Overview

SARS (Scholarly Architectural Reconstruction System) reconstructs the first-century BCE appearance of the Templum Divi Augusti in Ankara — a Corinthian peripteral temple dedicated to Augustus, partially preserved inside the Hacı Bayram Veli Mosque (1427 CE). The pipeline combines academic document retrieval, vision-language model analysis of 58 source images, and controlled image generation.

Pipeline

  1. Ingestion — 18 academic PDFs (primary, comparative, theory, local) are chunked with semantic boundary detection and stored in a ChromaDB vector store using sentence-transformers embeddings.
  2. Conditioning Prep — All 66 source images receive Canny edge extraction (OpenCV) and MiDaS depth estimation (Intel/dpt-large) where applicable. Filename-derived flags (minaret, scaffolding position, mosque) drive region-zeroing on the Canny maps at prep time.
  3. VLM Analysis — Each image is analysed with Gemma-3-4b-it. High-credibility sources undergo a two-pass strategy: a first pass generates a precise RAG query; a second pass re-analyzes with retrieved citation context. The output schema captures architectural elements, stratigraphic confidence, mosque-interference level, and an SDXL prompt component per image.
  4. Aggregation — Confirmed Roman elements are counted across all clean analyses (contaminated full shots, details, and inscriptions are excluded). Elements appearing in ≥2 independent sources or with high VLM confidence are promoted to the canonical prompt. Ottoman and mosque terms are stripped from every prompt component.
  5. Generation — SDXL base + ControlNet (canny) generates 1024×1024 reconstructions. Four canonical views are produced: front elevation, three-quarter, side elevation, and interior cella. Prompts are capped at 70 words to stay within CLIP's token window. All output is accompanied by a provenance JSON recording sources, citations, and generation parameters.

Contamination Control

The mosque presents a significant contamination challenge: its 1427 CE fabric is physically integrated with Roman masonry. Three strategies are applied:

  • Ottoman vocabulary is stripped from all VLM-extracted SDXL prompt components before generation.
  • Canny maps are cleaned at prep time using filename-derived position tokens — minaret regions zeroed, mosque walls trimmed, scaffolding masked per-position (top, middle-left, middle-right, etc.).
  • Parallel temples — Maison Carrée (Nîmes) and Temple of Augustus (Pula) — provide uncontaminated Corinthian geometry as additional conditioning sources.

Data Sources

  • Primary: 8 site-specific publications including Mitchell & Waelkens 1998 (DAI measured drawings), Hänlein-Schäfer 1985, Coulton 1976
  • Comparative: Parallel temple studies (Maison Carrée, Pula, Nîmes)
  • Theory: Vitruvius, Books III & IV (De Architectura)
  • Local: Ankara urban context and Ottoman period documentation
  • Visual: 66 images — full shots, architectural models, details, inscriptions, parallels, plans
